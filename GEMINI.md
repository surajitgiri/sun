# Project instructions

Always follow these rules unless I explicitly say otherwise:

- This is a Next.js project
- Never scan `node_modules` or `.next`
- Do not analyze `public` unless asked
- First analyze, do not edit files unless I say "fix"
- When reporting issues, always include exact file paths
- Report these separately:
  1. empty files
  2. incomplete components
  3. broken imports
  4. route issues
  5. likely build/runtime errors
- Keep output concise and structured
- Prioritize `src/app` and `src/components`
